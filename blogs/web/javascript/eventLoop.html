# 执行栈

总所周知 javascript 是单线程的，将只有一个调用栈,同一时刻只能做一件事情。也就是说代码只能一段一段地执行。看看一个代码示例:

function multiply(a, b) {
  return a * b;
}

function square(n) {
  return multiply(n, n);
}

function printSquare(n) {
  var squared = square(n);
  console.log(squared);
}

printSquare(4);

执行上面脚本时 将 printSquare(4)压入执行栈中, 然后此函数运行时, 又将 square(n) 压入栈中, 当执行 square(n)时, 再将 multiply(a,b)压入栈中, 执行完 multiply(a,b)函数后出栈, 然后执行完 square(n)出栈, 最后执行完 printSquare(4), 执行栈清空。

思考一下如下代码:

function foo() {
  return foo();
}

经上述讲解,显然会无限调用 foo(), 这样的话执行栈占用的空间将无限扩大, 此时浏览器会报错:

在真实场景显然不是这样函数的简单计算的, 我们遇到大量的图片加载、网络请求、大量数据循环等. 如果想上述那样的执行的显然是阻塞的, 影响浏览器页面渲染的。

# 任务队列

console.log("Hi");

setTimeout(function () {
  console.log("There");
}， 5000);

console.log("JSConfEU");

上述代码执行顺序是怎样的呢❓

答案

Hi > JSConfEU > There

你答对了吗？ 执行顺序为什么是这样呢？

TIP

步骤:

  1. 将 console.log("Hi")压入栈中执行，执行完后出栈
  2. 将 setTimeout 函数压入栈中执行, 执行中将 setTimeout 移入 webAPI 中,等待 5s 放入任务队列中
  3. 将 console.log("JSConfEU") 压入栈中执行, 执行完后出栈
  4. 等待五秒后将 setTimeout 的 callback 放入任务队列中
  5. 此时启动 event loop 机制 发现执行栈为空, 将 callback 放入执行中
  6. 执行 console.log("There")然后出栈

# 微任务 And 宏任务

我先看看一道题, 思考一下执行顺序

console.log(100);
setTimeout(() => {
  console.log(200);
});
Promise.resolve().then(() => {
  console.log(300);
});
console.log(400);
答案

100 400 300 200

如果你了解同步异步的话, 首先对先打印 100 和 400 是没有质疑的. 疑惑可能是先 300 然后打印 200. 这是因为异步执行分不同类型的任务. 有微任务和宏任务之分. 哪些是宏任务， 哪些是微任务呢？

  • 宏任务 setTimeout setInterval Ajax Dom 的监听事件 I/O

  • 微任务 Promise.then async/await MutationObserver

let btn = document.querySelector("#btn");
btn.addEventListener("click", function () {
  queueMicrotask(() => {
    console.log("microtask 1");
  });
  console.log(1);
});
btn.addEventListener("click", function () {
  queueMicrotask(() => {
    console.log("microtask 2");
  });
  console.log(2);
});
// 点击按钮btn元素 执行顺序是怎样呢？
答案

提示

在 Event Loop 时 会先检查微任务队列中有无任务, 有则将微任务队列中的任务放入执行栈中执行. 然后 Event Loop 会检查宏任务队列中有无任务, 有则将宏任务队列中的任务放入执行栈中执行.所以 setTimeout 要后 Promise.then 执行的原因

思考一下 如果不是点击按钮, 而是脚本执行 btn.click()又会是怎样的执行顺序呢❓

答案

1 > 2 > micotask 1 > microtask 2

你知道为什么吗？

原因

当通过 btn.click()执行时 会将宏任务打包成一个任务, 因此先执行 1 和 2 再执行微任务 micotask 1 microtask 2, 你也可以理解为将两个回调任务合并成一个回调任务

# 令人头疼的 This 对象

在 Javascript 中 this 动态指向，场景较多。 开发中经常会遇见 this 指向谁的问题。理解 this 对我们开发并且写出更加优雅的代码是极有裨益的。

# This 到底指向谁呢?

# 1. 全局环境中的 this

在全局环境中调用时，this 指向 window; 在严格模式下(use strict)指向 undefined

function fn() {
  console.log(this); // window
}
function strictFn() {
  "use strict";
  console.log(this); // undefined
}

这个是一个基础的列子, 下面已改打印什么呢?

const foo = {
  bar: 100,
  fn: funciton () {
    console.log(this)
    console.log(this.bar)
  }
}
const fn = foo.fn
fn()
答案

window undefined

原因

fon.fn 函数赋值给 fn 变量, 在执行 fn 这个变量函数时,执行环境在全局环境中。因此 this 为 window 对象,而 window 对象下没有 bar 属性, 所以打印 undefined

如果换成另外一种调用的话, 结果会是怎样呢？

const foo = {
  bar: 100,
  fn: funciton () {
    console.log(this)
    console.log(this.bar)
  }
}
foo.fn()
答案

foo 100

# 2. 上下文对象调用中的 this

当上下文对象调用时，this 指向最后调用它的对象。看看下面的例子，你知道答案吗？

const o1 = {
  text: 1,
  print() {
    return this.text;
  },
};
const o2 = {
  text: 3,
  print() {
    return o1.print();
  },
};
const o3 = {
  text: 3,
  print() {
    const print = o1.print;
    return print();
  },
};
答案

1, 1, undefined

原因

第一个打印为 1，比较简单且在上面提到就不赘述了

第二个打印也为 1 return 的是 o1 调用 print 方法, 对于 print 方法里面的 this 而言, 将会指向 o1 对象

第三个打印为 undefined 将 o1 的方法 print 赋值于变量 print，由于脱离了 o1 的执行上下文裸奔调用, 此时的 this 指向全局对象 window

bind、call、apply
